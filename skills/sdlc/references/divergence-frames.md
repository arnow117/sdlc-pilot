# divergence-frames — 可选「发散 ideation」pass(蒸馏自 adhd skill)

> distilled-from: adhd(MIT)。**蒸馏方法,不依赖外部 skill**:本文件是纯数据,
> 任何流程 skill(主要是 sdlc-spec 的方案探索、sdlc-build 的模糊调试)在**门控通过**时引用它跑一遍。
> 它是**发散**(多框架爆点子,禁显而易见),正好补 brainstorming/sdlc-spec 的**收敛**之不足。

---

## 0. 何时跑(pre-flight 门,三问全 yes 才跑)

这套**贵**(并行约 5-10 个 agent;串行更慢)。**默认不跑**,只在以下全满足时引入:

1. **开放式?** 资深工程师在此会给**多个**可行答案,而非唯一教科书答案。(唯一答案 → 不跑)
2. **高风险?** 选错"显而易见答案"的代价高:架构 / 公开 API 面 / 命名(真要发布的) / schema 设计 / 无已知根因的模糊 bug。(11点的小副项 → 不跑)
3. **开放措辞?** 用户没说 "quick / standard / canonical / textbook / just / 一行搞定"。(说了 → 用户要直接答案,不跑)

> 用户显式 `/adhd` 或"用发散/ADHD"= 无条件跑。否则按上面自判,任一为 no → **跳过,直接给常规答案**。

## 1. 两阶段(严格分离,混了就毁)

### Phase 1 — 发散(关掉评判)

1. 从下表选 **5 个认知框架**(代码型问题:4 个 code/design 标签 + 至少 1 个 wild;每次换着选,保证 range)。
2. **并行**起 5 个 agent(有 Task 能力)或**串行**逐框架跑(Codex/无并行)。每个 agent **只拿**:问题 P + 用户上下文 + 该框架的 vantage prompt + "禁止评判"指令。
   > 给每个 agent 的指令:「你在发散模式,是生成器不是批判者。在此框架下生成 6 个简短不同的点子,各一句话。不评判、不排序、不含糊。**前 3 个人人都会给的显而易见答案禁止出现**,往后面的尴尬中段推。只输出 JSON 数组。」
3. **铁律:并行且隔离**。不串行喂、不把一个分支的输出当另一个的上下文——分支互看就互相锚定,整个方法塌成"一个更宽的念头"。(串行降级时:每个框架**独立**起一段,不让后面的看前面的产出。)

### Phase 2 — 聚焦(打开评判)

1. **打分**:每个点子三轴 0-10 —— 新颖(离默认多远)/ 可行(真能落地)/ 契合(对不对题)。看着诱人但是**陷阱**的(隐藏成本/假经济/不可扩展/过早抽象)→ 标一行原因。
2. **聚类**:按**底层角度**(不是表面关键词)分 3-6 簇,簇名用角度命名(如"去掉服务器派""缓存型派""批窗口派")。
3. **深化 top 3**:按加权分(新颖 0.35 + 可行 0.40 + 契合 0.25)排序、排除陷阱、取前 3。每个深化出:4-8 句怎么运作的草图 + 承重风险 + builder 的第一步具体动作 + 3-5 个子点子(变体/杂交/解锁)。

## 2. 框架表(每次选 5)

| 框架 | vantage prompt(给 agent) | 标签 |
|---|---|---|
| **硬件工程师** | 用延迟/内存布局/物理约束想。把它重述成硬件/固件问题:总线拓扑、cache、时序预算告诉你什么? | code, wild |
| **监管者** | 审合规与失败模式。这里什么必须可证明/可追溯/可拒绝? | design, general |
| **10 岁小孩** | 没见过软件的好奇小孩。无视惯例,描述天真但无包袱的做法。 | general, wild |
| **想搞垮它的竞争者** | 敌对竞争者/攻击者。生成利用/搞砸/破坏显而易见方案的做法,再反转成点子。 | code, design |
| **生物** | 移植一个生物机制(免疫/神经可塑/细胞信号/进化/肠道菌群)硬套到此工程问题。 | code, wild |
| **物流** | 偷物流机制:队列、批处理、JIT、枢纽辐射、退货、最后一公里。直接套用。 | code, design |
| **inversion 反转** | 问相反的问题:目标是 X,就头脑风暴怎么**保证 NOT X**,再把每个答案取反。 | code, design, general |
| **移除承重假设** | 点名大家当固定的东西(框架/数据库/请求-响应模型/网络),假设它没了。还能怎么做? | code, design, wild |
| **$0 预算 / 1 小时** | 没钱没团队一小时。还能做承重那件事的**最粗版**是什么? | code, general |
| **无限预算 / 10 年** | 无限算力工程师十年。**最大主义**版本是什么?(雄心/做大方向) | design, wild |
| **speedrunner** | 找 glitch、skip、越界、帧完美捷径。**滥用但合法**的路径是什么? | code, wild |
| **3 点被叫醒的 on-call** | 半夜被它叫醒的值班工程师。什么设计能让你**不被 page**? | code, design |
| **蚁群** | 无中央规划,许多笨 agent + 局部规则 + 信息素。问题怎么**涌现式**自解? | code, wild |

## 3. 输出形状(别糊成一坨散文,结构本身是重点)

1. **Brief** — 一两行确认问题 + 用了什么 reframe。
2. **Wide set** — 全池按簇分组,每簇标底层角度,每点一短语,带分数 chip `[N7 V8 F9]`。
3. **Converge** — 2-4 个短名单,各说为何入选。**用 ★ 显式标"非显而易见但可行"那个**;陷阱单列,各一行原因。
4. **Focus** — 3 个深化分支:草图 / 承重风险 / 第一步 / 子点子。
5. **Provocation** — 一个 wildcard,留一条新方向给用户继续推。

## 4. 在 sdlc 流程里的接入点(谁在何时引用本文件)

- **sdlc-spec §2.4**(主):问题已理解(Explore/Clarify 完)、要选方案前 → 过本文件 §0 门 → 通过则先跑发散,把存活的非显而易见 survivors **喂进**收敛的 2-3 方案;不通过则直接给常规 2-3 方案。
- **sdlc-build 调试**:模式签名表**没命中**的模糊 bug → 可选用本文件出"假设类",再回 Phase 3 逐一最小验证。
- 接入永远是**可选 + 门控 + Task-or-sequential**,不是默认步,不破可移植。
